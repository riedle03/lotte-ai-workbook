# 18. 2차시 슬라이드 설계도 — PPT 제작용 상세본

> **버전**: v2 (상세 레이아웃 포함)
> **강사 시연**: 가을 동남아 가족 기획전 → Tona / 오사카 단풍 → 카피 마스터
> **실습 힌트 직군**: 영업 / 예약·발권 / 경영지원·인사총무
> **총 슬라이드**: 22장 (표지 1 + 섹션 디바이더 2 + 콘텐츠 19)
> **표기법**: 헤드라인=H / 서브타이틀=S / 본문=B / 강사멘트=M / 레이아웃=L

---

## 핵심 반복 구조

```
【전반 루프 — Tona】
★문제(S3) → 기획방향성 마스터 소개(S4) → 시연1 목차(S5) →
Tona 원리(S6) → Tona 전문(S7) → 시연2 보고서(S8) →
Before/After(S9) → ★실습(S10) → 힌트(S11) → 보너스(S12)

【전환 디바이더(S13)】

【후반 루프 — 카피 마스터】
★문제(S14) → 카피 마스터 소개(S15) → 전문(S16) →
시연 카피(S17) → ★실습(S18) → 힌트(S19) → 보너스(S20)

【마무리】 체크인(S21) → 3차시 예고(S22)
```

### S3 — ★ 문제 제기: "내용은 같은데 왜 반응이 다를까?"

**L**: Pattern E — Quote + Evidence

**H**: `기획서는 완성했는데 — 왜 팀장과 본부장 반응이 다를까`

**S**: `문서 두 개를 만드는 게 문제가 아니다. 뭘 쓸지 모른 채 어떻게 쓸지부터 고민하는 것이 문제다`

**B 좌열 (6.5" 너비) — 인용 블록**:
- 인용 기호 (Bold 48pt, #024ad8): `"`
- 인용문 (Medium 22pt, #1a1a1a, 줄간 1.3):
  ```
  김 과장은 기획서의 스토리라인을
  완벽하게 정리했다.
  그런데 진짜 고민은 이제부터 시작된다.
  보고를 받는 팀장과 본부장의
  스타일이 완전히 다르기 때문이다.
  '그럼 두 건의 문서를 만들어야 하는 건가…?'
  ```
- 출처 (Regular 12pt, #636363): `— 책 p.89`

**B 우열 (4" 너비) — 3개 진단 카드 (세로 적층, #f7f7f7, radius 16px)**:

카드 1:
- 타이틀 (SemiBold 13pt): `커서가 깜빡인다`
- 내용 (Regular 11pt): 기획서를 열었는데 첫 줄을 못 쓰는 상황. 뭘 써야 할지 몰라서.

카드 2:
- 타이틀 (SemiBold 13pt): `Tona를 켰다`
- 내용 (Regular 11pt): 어떻게 쓸지 도구는 있는데 — 내용이 없다. 순서가 거꾸로다.

카드 3:
- 타이틀 (SemiBold 13pt, #024ad8): `순서가 있다`
- 내용 (Regular 11pt, #024ad8): **뭘 쓸지 먼저 → 어떻게 쓸지 나중**

**M**:
> "여러분도 이런 경험 있죠? 기획서 열고 커서 깜빡이는데 — 뭐부터 써야 할지 모르겠는 거예요.
> 이 상황에서 Tona 쓰면 어떻게 될까요? 아무것도 안 나옵니다.
> 순서가 있어요. '뭘 쓸지' 먼저, '어떻게 쓸지' 나중입니다."

### S4 — 해결책: 기획방향성 마스터 + Tona 2단계 흐름

**L**: Pattern D — Process Flow (5단계 → 화살표 → Tona)

**H**: `기획방향성 마스터가 목차를 만들고 Tona가 보고서로 포맷한다`

**S**: `사고 코치(뭘 쓸지) + 톤 엔지니어(어떻게 쓸지) — 2단계 연결`

**B — 가로 플로우 다이어그램**:

상단 레이블 (SemiBold 14pt, #636363): `기획방향성 마스터 — STEP 1~5`

5개 단계 원형 노드 (파란 원 #024ad8, 흰 숫자 Bold 18pt):
```
① 과제정보  →  ② ASK분석  →  ③ 전략도출  →  ④ 질문세트  →  ⑤ 목차자동
지시자/결정자   보고 대상 파악   접근 방향 확정   사고 흐름 정리   논리 목차 완성
```

큰 화살표 (→, #024ad8) + 레이블 (SemiBold 14pt):
```
이 목차를 Tona에 넘긴다
```

Tona 카드 (#1a1a1a, radius 16px, 우측):
- 카드 타이틀 (SemiBold 14pt, #fff): `Tona — 톤 엔지니어`
- 내용 (Regular 12pt, #fff 70%):
  - ① 목적  ② 독자  ③ 길이
  - ④ 상단요약  ⑤ 소제목  ⑥ 본문
  - → 보고서·기획안 완성

**하단 So What** (#c9e0fc):
```
기획방향성 마스터 = "AI가 먼저 질문하게 만드는 사고 코치"
```

**M**:
> "기획방향성 마스터는 답을 내는 도구가 아닙니다.
> 제가 답하면 AI가 목차를 만들어줘요. 그 목차를 Tona에 넘기면 — 보고서가 됩니다.
> 2단계입니다. 빠르게 시연 보여드릴게요."

### S5 — 강사 시연 1: 기획방향성 마스터 → 목차 도출

**L**: Pattern B — 2열 (좌: 입력값 / 우: AI 출력 목차)

**H**: `가을 동남아 가족 기획전 — STEP 1~2 입력, 목차 5개 자동 도출`

**S**: `시연: 상품기획팀 기획안 · 보고 대상 영업본부장`

**B 좌열 — 입력값 카드** (#f7f7f7, radius 16px):
- 타이틀 (SemiBold 14pt): `강사 입력값`
- 내용 (Regular 12pt, 줄간 1.5):
  ```
  STEP 1. 과제 기본정보
  지시자:      상품기획 팀장
  의사결정자:  영업본부장
  과제명:      2026 가을 동남아 가족 기획전 콘셉트 기획안

  STEP 2. ASK 분석
  Attitude:   ② 성과·효율 중심
              ⑤ 전략·방향성 중시
  Style:      ① 결론 먼저
              ② 숫자·지표 중심
  Knowledge:  ① 이슈에 익숙함
  ```

**B 우열 — AI 출력 목차** (카드, #ffffff, 테두리 #024ad8 1px):
- 타이틀 (SemiBold 14pt, #024ad8): `AI가 도출한 목차 (5개)`
- 내용 (Regular 12pt):
  ```
  1. 추진 배경 및 현황
     왜 지금 가을 동남아 가족 기획전인가

  2. 타깃 고객 분석
     누구에게, 어떤 가치를 팔 것인가

  3. 콘셉트 및 차별화 전략
     경쟁 상품과 무엇이 다른가

  4. 상품 구성 및 가격 전략
     무엇을, 어떻게, 얼마에 팔 것인가

  5. 실행 계획 및 기대 효과
     언제, 어떻게, 얼마나
  ```
- 하단 화살표 레이블 (SemiBold 12pt, #024ad8):
  `이 목차 → Tona에 복사·붙여넣기`

**M**:
> "STEP 2에서 보고 받는 분의 스타일을 골라주면 AI가 그 사람에게 맞는 목차 구조를 뽑아줍니다.
> '결론 먼저 + 숫자 중심'을 선택했더니 목차가 그 방향으로 설계됐죠.
> 이 목차가 Tona의 '내용' 역할을 합니다."

### S6 — Tona 원리: 6단계 선택 + 8대 원칙

- (Regular 15pt, #3d3d3d, 중앙):

  ```
  "좋은 표현이란 상대의 입장에서 핵심을 즉시 파악할 수 있는 표현이다."  — 책 p.90
  ```

**B 중단 — 6단계 선택표** (테이블, y=3.05"~5.3"):

| # | 질문 | 선택지 |
|---|---|---|
| ① | 문서 목적 | 보고 / 제안 / 분석 / 기획 / 계획 |
| ② | 문서 독자 | 팀장 / 임원 / CEO / 공공기관 담당자 |
| ③ | 문서 길이 | 1페이지 / 2페이지 / 3페이지 / PPT 3장 |

| ⑥ | 본문 스타일 | 간결형 / **전략메시지형(Fact+해석+시사점)** |

- 테이블 헤더: #1a1a1a bg, #fff 텍스트
- 굵게 표시된 항목: 시연에서 선택할 값

**B 하단 — 8대 원칙 So What** (#c9e0fc, y=5.4"~6.85", radius 8px):
- 레이블 (SemiBold 13pt, #024ad8): `Tona 8대 문장 원칙`
- 내용 (Regular 12pt, #024ad8, 2열 4행):

  ```
  감정·미사여구 제거        핵심 명사 중심 짧은 문장
  숫자·지표 반드시 병기(%)  화살표(→)로 흐름 구조
  ```

### S8 — 강사 시연 2: Tona → 기획안 보고서 출력
