# 17. 1차시 슬라이드 설계도 (책 텍스트 기반)

> 용도: PPT 제작 전 슬라이드 구성 설계
> 강사 시연 고정: 상품기획 (오사카 단풍 패키지)
> 실습 힌트: 3개 대표 직군 (영업 / 예약 / 경영지원)
> 각 슬라이드: 화면에 보여줄 것 + 강사 멘트 + 출처 명시

---

## 핵심 반복 구조

```
문제 제기 → 해결책 소개 → 강사 시연 → 수강생 실습 → 실습 힌트
```

**슬라이드 2 — 여러분도 이런 경험 있지 않으신가요?**

- 화면: 책 p.66 인용 박스 3개 나란히

  > "AI가 대답은 잘하는데… 내가 원하는 결과가 안 나와."
- 하단: → 결국 "내 일은 AI로는 못하지 뭐" → **조용한 안도감** → 그냥 하던 대로

- 강사 멘트:
  > "AI 써보셨죠? 근데 막상 해보면 이런 생각 드셨을 거예요.
  > 이게 딱 책에 나온 말인데요 — 대부분의 직장인이 비슷한 좌절을 겪는다고요.

**슬라이드 3 — AI가 들어오면 가장 먼저 사라지는 것**

  | 상황 | 결과 |
  |---|---|

- 하단 강조 박스:
  > **"준비 없이 맞이하면, 가장 먼저 사라지는 건 업무량이 아니라 사고력이다"**
  > — 책 p.67

- 강사 멘트:
  > "AI가 결과물을 대신 써줘서 좋았는데, 어느 순간 내가 그 내용을 설명을 못하겠는 거예요.
  > 이 상태가 되면 책은 이렇게 표현합니다.
  > '당신은 AI에 실려 이동된 화물일 뿐이다'라고요.
  > 오늘 우리가 배울 건, 화물이 아니라 **기수**가 되는 법입니다."

**슬라이드 9 — 사고 설계형의 설계도: ROADS**

- 화면: 책 p.75-76 내용 기반 표

  | 원칙 | 뜻 | 책의 설명 |
  |---|---|---|
  | **역** (Role) | 역할을 부여하라 | "AI는 가면을 씌울 때 실력이 달라진다" |
  | **목** (Objective) | 목적을 선명히 하라 | "'OO 만들어줘'는 지시가 아니라 떠넘기기다" |
  | **길** (Approach) | 길을 먼저 물어라 | "답보다 길을 먼저 물으면 AI는 파트너가 된다" |
  | **단** (Drive step) | 단계적으로 함께하라 | "목차 → 초안 → 내용, 한 번에 완벽을 기대 마라" |
  | **다** (Sharpen) | 다듬어 완성하라 | "'5%만 더 날카롭게' — AI 결과물은 언제나 초안" |

- 책 직접 인용:
  > **"프롬프트는 'AI 스킬'이 아니라 '일잘러의 사고를 담은 설계도'이다"**
  > — 책 p.76

- 강사 멘트:
  > "ROADS 기억하시겠어요? 역목길단다.
  > 그런데 이걸 매번 머릿속에서 직접 짜려면 힘들잖아요.
  > 그래서 책이 준비한 게 있습니다."

  > "이게 왜 어렵냐고 책이 설명해요.
  > 우리는 인간 언어를 쓰고, AI는 AI 언어를 써요.
  > 매번 번역 작업을 해야 하니까 힘든 거예요."

**슬라이드 12 — 해결책: 통역사를 세워라**

- 화면: 책 p.79 해결책 인용

  > "영어를 못할 때 가장 효율적인 해결책은
  > 단어 몇 개를 외우는 게 아니라 **유능한 통역사를 활용하는 것**이다.
  > 프롬프트도 똑같다.
  > AI에게 '프롬프트를 만드는 역할'을 시키는 것이 더 효율적이다."
  > — 책 p.79

- 프롬프트 마스터 구조 한눈에:
  1. Context (상황) → 2. Objective (목적) → 3. Constraints (제약) → 4. Facts (사실) → 5. Target (대상)
  - "이 5가지에 답하면 → 실행 프롬프트 자동 생성"

- 강사 멘트:
  > "제가 5개 질문에만 답하면, AI가 제대로 된 프롬프트를 대신 짜줍니다.
  > 이게 프롬프트 마스터입니다. 지금 바로 보여드릴게요."

**슬라이드 14 — 강사 시연: 오사카 단풍 패키지 런칭 공지문**

- 화면: 강사 Gemini 화면 미러링 + 입력값 옆에 표시

  2️⃣ Objective: 영업·예약팀에게 상품 개요·특장점·판매 시작일 공유
  3️⃣ Constraints: A4 반 장 이내 / 불릿 형식 / 과장 표현 없이
  4️⃣ Facts: 오사카·교토 5일 / 11월 7일 출발 / 1,290,000원 /
             새벽 아라시야마 + 나라 야간 단풍 포함 / 선착순 28명
  5️⃣ Target: 영업팀·예약팀 실무자 (내부 공지 수신자)
  ```

- 강사 멘트:
  > "ROADS 원칙을 제가 직접 설계하지 않아도
  > 프롬프트 마스터가 5개 질문으로 구조를 잡아줍니다.

**슬라이드 17 — 실습 힌트: 3개 직군 입력 예시**

- 화면: 3개 직군 입력값 (참고용, 화면 고정)

  **💼 영업·판매 — 신상품 거래선 세일즈 포인트 정리문**
  ```
  1️⃣ Context:  오사카 단풍 신상품 출시 → 여행사 담당자 설득용 자료
  2️⃣ Objective: 핵심 세일즈 포인트 3개 + 차별점 정리 (A4 반 장)
  3️⃣ Constraints: 불릿 형식 / 경쟁 상품 대비 우위 포함 / 과장 없이
  4️⃣ Facts: 오사카·교토 5일 / 11.7 출발 / 129만원 /
             새벽 아라시야마 단독 입장 / 선착순 28명
  5️⃣ Target: 거래선 여행사 영업 담당자

  |---|---|
  | 범용 — 어떤 문서든 | 메일 특화 |

- 메일 마스터 7단계 한눈에:

  ```
  1️⃣ 수신 정보 (To/CC) — 읽을 사람·참조할 사람
  2️⃣ 메일 목적 — 이 메일을 쓰는 근본적인 이유
  ★ 3️⃣ 수신자의 예상 질문 — 읽는 사람이 가장 궁금해하거나 우려할 것 2~3가지
  4️⃣ 본문 핵심 내용 — 3번 질문에 대한 답 포함
  5️⃣ 기대 행동 — 메일 읽고 즉시 해줬으면 하는 것
  6️⃣ 톤 선택 — 격식+친절 / 협조+실무 / 간결+보고
  7️⃣ 첨부 자료
  ```

- 강조 박스:
  > **"기존 메일 자동화는 '내가 전달하고 싶은 것' 중심.**
