# 13. 4차시 흐름 정리

> 차시 재설계 v2 — 책 5장 리사 + 책 4장 IFE + Gemini 데이터 분석
> 세부 시연 시나리오는 별도 문서, 이 파일은 흐름만

---

## 4차시 핵심 메시지

> **"1·2·3차시는 글·콘텐츠·자료 창고를 만들었다.
> 마지막 차시는 숫자를 말이 되는 보고서로 — 4시간의 종착역.
> 그리고 워크북 4칸을 채우고 다음 주 시도할 1가지를 손에 들고 나간다."**

**손에 들고 나갈 산출물 1세트 + 자기선언**:
1. 본인 직군 **더미 데이터 분석 + 보고용 문장 + PPT 슬라이드 5장 목차** (메일 ⑥⑦ 통합 산출)
2. **워크북 4칸 완성 + 다음 주 시도할 1가지** (4시간 마무리 의식)

**핵심 도구**: Gemini 데이터 분석 + 리사 (책 5장)

| 블록 | 시간 | 핵심 |
|---|---|---|
| A. 오프닝 | 3분 | 3차시 회고 + "마지막은 보고 — 숫자를 말이 되는 보고서로" |
| B. 다리 놓기 | 4분 | 1~3차시 누적 회수 + "보고가 마지막 관문" |
| C. ① Gemini 데이터 분석 시연 | 9분 | CSV 업로드 → 인사이트 3줄 (메일 ⑥ 직접) |
| D. ② 리사 시연 | 8분 | C 인사이트를 리사 답변으로 → 보고 멘트 + 핵심 요약 + PPT 목차 (메일 ⑦ 직접) |
| E. 실습 | 13분 | 본인 직군 더미 데이터로 같은 흐름 (분석 → 리사) |
| F. 종합 자기선언 + 마무리 | 13분 | 워크북 4칸 + 부서별 1명 30초 발표 + 자율 학습 로드맵 |

→ 합 50분

## A. 오프닝 (3분) — "마지막은 보고"

- 3차시 체크인 한 명 호명 ("부서에 가장 필요한 것") (1분)
- 이번 차시 성격 선언 (2분):
  - "1·2·3차시는 글·콘텐츠·자료 창고를 만들었다"
  - "이번 차시는 그걸 **숫자와 함께 윗사람에게 전달하는 마지막 관문**"
  - "끝에 산출물 1세트 + 다음 주 시도할 1가지를 손에 들고 갑니다"

---

## B. 다리 놓기 (4분) — "보고가 마지막 관문"

1. **1~3차시 누적 회수** (2분)
   - 1차시 = 정형 비즈니스 문서 (메일·기안)
   - 2차시 = 보고서·기획안 (Tona) + 카피 (카피 마스터)
   - 3차시 = 부서 데이터 창고 (NotebookLM)
   - **4차시 = 그 모든 결과를 임원·상사에게 보고하는 것**

2. **책 5장의 핵심 통찰** (2분)
   - 책: *"보고는 보고자가 할 말이 아니라, 상사가 던질 질문에 답하는 것"*
   - → 1차시 메일 마스터 "수신자 예상 질문(★)"의 보고 확장판
   - → 책의 리사(ReSa) 프롬프트 도입 예고

---

## C. ① Gemini 데이터 분석 시연 (9분) — 메일 ⑥

1. Gemini에 더미 CSV 업로드 (1분)
   - 예: 분기 상품별 매출 데이터 (지역·상품·매출·이탈률)
2. 자연어 분석 시연 (5분)
   - *"이 데이터에서 임원 보고용 인사이트 3개 뽑아줘"*
   - 결과 출력: 인사이트 3줄 + 근거 데이터
3. 추가 질문 시연 (2분)
   - *"가장 우려되는 리스크 1개는?"* → 추가 인사이트
4. "이게 ⑥ 엑셀 요약 — 다음은 이걸 보고서로" 정리 (1분)

→ **산출물 (시연용)**: 인사이트 3줄

---

## D. ② 리사 시연 (8분) — 메일 ⑦

1. 리사 전문 붙여넣기 + 입력 시작 (2분)
   ```
   1) 보고 주제: 분기 베스트 상품 인사이트 보고
   2) 보고 대상: 본부장
   3) 보고 방식: 5) PPT
   ```
2. 리사 자동 진단 노출 (1분)
   - "이 보고는 **현황 공유형**입니다. 스토리라인은 **PREP**, PPT 방식 기준 질문 **5~8개**가 적절합니다."
3. 상사 질문 라이브 답변 (4분)
   - 리사가 4~5개 질문 던짐 ("지금 한 줄로 말하면?" / "근거는?" / "리스크는?" / "다음 단계는?")
   - C 블록 인사이트 3줄을 답변으로 입력
4. 산출물 출력 (1분)
   - 구두 보고 멘트 + 핵심 요약 3줄 + **PPT 5장 목차**

→ **산출물 (시연용)**: 보고 멘트 + 핵심 요약 3줄 + PPT 슬라이드 5장 목차

---

## E. 실습 (13분) — 본인 직군 데이터로 분석 → 리사 한 흐름

| 단계 | 시간 | 활동 |
|---|---|---|
| 1 | 1분 | 직군별 더미 CSV 다운로드 (강사 사전 배포) |
| 2 | 5분 | Gemini에 업로드 → 인사이트 3줄 뽑기 |
| 3 | 5분 | 리사에 보고 주제·대상·방식 입력 → 인사이트로 답변 → 보고 산출물 |
| 4 | 2분 | "수정하거나…" 피드백 한 번 돌리기 |

### 직군별 4차시 실습 시나리오

| 직군 | 더미 데이터 | 리사 보고 유형 × 방식 |
|---|---|---|
| 상품기획·오퍼레이터 | 분기 상품별 판매 데이터 | 현황 공유형 × PPT |
| 영업·판매 | 거래선별 매출 데이터 | 현황 공유형 × 1~2분 구두 |
| 예약·발권 | 노선별 예약률 데이터 | 판단 요청형 × 문서(A4) |
| 경영지원·인사총무 | 부서별 예산 집행 데이터 | 판단 요청형 × PPT |
| 고객서비스·VOC | 월간 VOC 카테고리 데이터 | 현황 공유형 × 1~2분 구두 |
| 카지노·VIP | VIP 등급별 이용 데이터 | 기획·전략형 × PPT |
| IT·시스템 | 시스템 장애·문의 데이터 | 판단 요청형 × 문서(A4) |

→ **4차시는 전 직군 가치 회수 차시** — 7직군 모두가 자기 데이터로 자기 보고 패턴 체험

## F. 종합 자기선언 + 마무리 (13분)

| 시간 | 활동 |
|---|---|
| 0:00~3:00 | **워크북 4칸 채우기** (1·2·3차시 체크인 회수 + 4차시 다음 주 시도할 1가지) |
| 3:00~3:30 | 옆 사람과 30초 공유 |

| 10:00~13:00 | **자율 학습 로드맵 슬라이드** (Level 1~5) + 강의 종료 인사 |

```
[4차시 종합] 다음 주에 시도할 1가지: ______________________
```

## 메시지 누적 흐름 (한눈에)

```
A. 3차시는 자료 창고 → 이번 차시는 그 위에 숫자를 얹어 보고서까지
        ↓
B. 1~3차시 누적 산출물 회수 → 마지막 관문 = 보고
        ↓
B. 책 5장 통찰: "보고는 상사가 던질 질문에 답하는 것" (메일 마스터 ★의 보고 확장판)
        ↓
C. Gemini 데이터 분석 — CSV → 인사이트 3줄 (⑥)
        ↓
D. 리사 — C 인사이트를 답변으로 → 보고 멘트 + PPT 목차 (⑦)
        ↓
E. 본인 직군 데이터로 직접 (분석 → 리사 한 흐름)
        ↓
F. 워크북 4칸 + 부서별 1명 30초 발표 + 자율 학습 로드맵
```

|---|---|---|

## 4차시의 두 가지 정체

**정체 1. 마지막 산출물 차시**
- 메일 ⑥⑦ 직접 처리, 분석→보고 한 흐름

**정체 2. 4시간 전체의 마무리 의식**
- 워크북 4칸 누적 회수 + 다음 주 시도할 1가지 공개 선언
- 부서별 1명 30초 발표로 동료에게 약속 → follow-up 동력
- 자율 학습 로드맵으로 강의 후 길 안내

---

## 결정 필요한 항목

1. **더미 데이터팩** — 7직군 × CSV 1개씩 사전 준비 (강사 작업)
   - 3차시 NotebookLM 자료팩과 통합 가능 (같은 직군은 같은 더미 상품 기준으로 일관성)
2. **C 시연 더미 데이터** — 분기 베스트 상품 매출 데이터 (지역·상품·매출·이탈률) 더미 1개 우선 작성
3. **부서별 30초 발표 운영** — 직군 7개 중 일부만 발표 가능 (실제로는 5~6명). 강사가 사전에 발표자 지명 vs 자원 받기 결정 필요
