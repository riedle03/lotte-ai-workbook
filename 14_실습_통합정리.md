# 14. 실습 통합 정리 (1~4차시)

> 차시별 실습 블록(E블록)과 직군별 시나리오를 한 파일에 통합
> 강사 시연용 더미 시나리오 제작 시 이 파일을 기준으로

---

## 실습 전체 한눈에 보기

| 차시 | 실습 블록 | 시간 | 사용 프롬프트 | 도구 | 산출물 |
|---|---|---|---|---|---|
| 1차시 | E | 12분 | 프롬프트 마스터 | Gemini | 정형 비즈니스 문서 1건 (A4 1장 이하) |
| 2차시 전반 | C | 11분 | 기획 방향성 마스터 → Tona | Gemini | 보고서·기획안 1건 (A4 1~3장) |
| 2차시 후반 | E | 11분 | 카피 마스터 | Gemini | 광고 카피·홍보 문구 1건 |
| 3차시 | E | 16분 | — (NotebookLM UI) | NotebookLM | 부서 노트북 1개 + 세일즈 포인트 요약 |
| 4차시 | E | 13분 | — (Gemini 자연어 + 리사) | Gemini + 리사 | 인사이트 3줄 + 보고 멘트 + PPT 목차 1세트 |

> **기획 방향성 마스터**: 2차시 전반 Tona 실습의 1단계 진입 프롬프트. 사고 흐름 정리 + 목차 도출 → Tona로 보고서 포맷하는 2단계 구조.

## 1차시 실습 (E블록 · 12분)

### 기본 정보

| 항목 | 내용 |
|---|---|
| 도구 | Gemini (무료) |
| 프롬프트 | **프롬프트 마스터** — 워크북 인쇄본 제공 |
| 방식 | 5단계 질문 답변 → 실행 프롬프트 자동 생성 → 결과 생성 → 피드백 1회 |
| 산출물 | 본인 직군 **정형 비즈니스 문서 1건** (A4 1장 이하) |

### 직군별 시나리오

| 직군 | 실습 과제 | 문서 유형 |
|---|---|---|
| 상품기획·오퍼레이터 | 신상품 런칭 내부 공지 | 공지문 |
| 영업·판매 | 거래선 방문 결과 보고 | 업무 보고 메일 |
| 예약·발권 | 항공권 규정 변경 고객 안내 | 고객 안내문 |
| 경영지원·인사총무 | 부서 워크숍 기안문 *(강사 시연 시나리오)* | 그룹웨어 기안문 |
| 고객서비스·VOC | 고객 불만 처리 결과 안내 메일 | 대외 메일 |
| 카지노·VIP | VIP 이벤트 초청 안내문 | 안내문 |
| IT·시스템 | 시스템 점검 공지 | 사내 공지 |

### 실습 흐름

```
1. 워크북 프롬프트 마스터 전문 확인 (30초)
2. Gemini에 프롬프트 마스터 붙여넣기 → "프롬프트 마스터, 시작해줘" (1분)
3. 5단계 질문에 본인 직군 시나리오로 답변 (5분)
4. 자동 생성된 실행 프롬프트로 결과 생성 (3분)
5. "수정하거나 보완하고 싶은 부분이 있나요?" 피드백 1회 (2분 30초)
```

### 숙련자 보너스 (5분 경과 시 공개)

- 같은 시나리오에 **메일 마스터** 적용 → 결과 비교
- 또는 **Claude**로 같은 작업 → 한국어 톤 차이 체감

## 2차시 실습 (전반 C블록 + 후반 E블록 · 각 11분)

### [전반] Tona 실습 (C블록 · 11분)

| 항목 | 내용 |
|---|---|
| 도구 | Gemini (무료) |
| 프롬프트 | **(기획 방향성 마스터 →) Tona** — 워크북 인쇄본 제공 |
| 방식 | Tona 6단계 선택값 입력 → 보고서·기획안 출력 |
| 산출물 | 본인 직군 **보고서·기획안 1건** (A4 1~3장) |

#### 직군별 시나리오

| 직군 | 실습 과제 | 문서 유형 |
|---|---|---|
| 상품기획·오퍼레이터 | 신상품 기획안 | 기획서 |
| 영업·판매 | 거래선 협의 결과 보고서 | 보고서 |
| 예약·발권 | 분기 예약 현황 보고 | 현황 보고 |
| 경영지원·인사총무 | 부서 운영 보고 / 사업 제안 | 보고서 또는 제안서 |
| 고객서비스·VOC | VOC 월간 분석 보고 | 분석 보고서 |
| 카지노·VIP | VIP 등급별 운영 보고 | 운영 보고서 |
| IT·시스템 | 시스템 운영 현황 보고 | 현황 보고 |

#### 실습 흐름

```
1. 워크북 Tona 전문 확인 (30초)
2. Gemini에 Tona 붙여넣기 (1분)
3. Tona 6단계 선택값 입력 (3분)
   ① 목적 / ② 독자 / ③ 길이 / ④ 상단 요약 여부 / ⑤ 소제목 스타일 / ⑥ 본문 스타일
4. 결과 출력 + 확인 (4분)
5. "이 부분을 좀 더 구체적으로" 피드백 1회 (2분 30초)
```

### [후반] 카피 마스터 실습 (E블록 · 11분)

| 항목 | 내용 |
|---|---|
| 도구 | Gemini (무료) |
| 프롬프트 | **카피 마스터** (강사 자작) — 워크북 인쇄본 제공 |
| 방식 | 7단계 선택값 입력 → 광고 카피 3안 출력 → 1안 선택·정교화 |
| 산출물 | 본인 직군 **광고 카피·홍보 문구 1건** (메인+서브+본문 3안 중 선택) |

#### 직군별 시나리오

| 직군 | 실습 과제 | 카피 유형 |
|---|---|---|
| 상품기획·오퍼레이터 | 신상품 홍보 카피 또는 기획전 카피 | 홍보 문구 |
| 영업·판매 | 거래선용 세일즈 한 줄 | 세일즈 카피 |
| 예약·발권 | 고객용 프로모션 안내 카피 | 홍보 카피 |
| 경영지원·인사총무 | 사내 캠페인 슬로건 | 캠페인 카피 |
| 고객서비스·VOC | 고객 감사 카드 한 줄 | 감성 카피 |
| 카지노·VIP | VIP 이벤트 초청 카피 | 프리미엄 카피 |
| IT·시스템 | 시스템 런칭 카피 (해당 시) / 또는 Tona 보완으로 시간 활용 | 안내 카피 |

#### 실습 흐름

```
1. 워크북 카피 마스터 전문 확인 (30초)
2. Gemini에 카피 마스터 붙여넣기 (1분)
3. 7단계 선택값 입력 (4분)
   ① 상품/서비스 / ② 핵심 메시지 1줄 / ③ 타깃 / ④ 카피 유형
   ⑤ 분량 / ⑥ 금기 표현 / ⑦ 참고 예시(선택)
4. 3안 출력 → 가장 마음에 드는 1안 선택 (3분)
5. "이 안을 좀 더 강렬하게" 피드백 1회 (2분 30초)
```

#### 숙련자 보너스

- 같은 카피를 **Claude**로 다시 → 한국어 문체 차이 비교
- 선택한 카피를 ChatGPT(이미지 생성) 또는 캘리그라피 앱으로 시각화 (회신 요청 반영, 방법 안내만)

## 3차시 실습 (E블록 · 16분)

### 기본 정보

| 항목 | 내용 |
|---|---|
| 도구 | NotebookLM (Google 계정 필요) |
| 프롬프트 | 없음 — NotebookLM UI 자체 |
| 방식 | 직군별 더미 자료팩 업로드 → 첫 질문 → 결과 정리 → 마인드맵/오디오 개요 체험 |
| 산출물 | 본인 부서 **NotebookLM 노트북 1개** + **세일즈 포인트·자료 요약 1건** |
| 사전 준비 | 강사가 직군별 더미 자료팩(3~5건) 사전 배포 |

### 직군별 시나리오

| 직군 | 업로드할 더미 자료 (3~5건) | 뽑을 결과물 | 첫 질문 예시 |
|---|---|---|---|
| 상품기획·오퍼레이터 | 상품 매뉴얼·전년 기획안·고객 후기 | **세일즈 포인트 5개** | "이 상품의 핵심 세일즈 포인트 5개를 뽑아줘" |
| 영업·판매 | 거래선별 협의 기록·계약서 샘플 | 거래선 특성 요약 | "이 거래선의 특성과 주요 협의 사항을 요약해줘" |
| 예약·발권 | 노선·요금·규정 문서 | 변경·취소 규정 FAQ | "자주 묻는 변경·취소 관련 질문과 답변을 정리해줘" |
| 경영지원·인사총무 | 사내 규정·복무·재무 매뉴얼 | 복무 규정 핵심 정리 | "복무 규정에서 직원이 꼭 알아야 할 핵심 5가지를 뽑아줘" |
| 고객서비스·VOC | 과거 VOC 사례·모범 답변집 | 유사 사례 답변 패턴 | "유형별 VOC 대응 패턴을 정리해줘" |
| 카지노·VIP | VIP 응대 매뉴얼·등급 가이드 | 등급별 응대 체크리스트 | "VIP 등급별 응대 체크리스트를 만들어줘" |
| IT·시스템 | 시스템 매뉴얼·트러블슈팅 기록 | 자주 묻는 오류 FAQ | "자주 발생하는 오류와 해결 방법을 FAQ로 정리해줘" |

### 실습 흐름

```
1. 강사 배포 직군별 더미 자료팩에서 본인 직군 선택 (1분)
2. NotebookLM 노트북 신규 생성 (1분)
3. 자료 3~5건 업로드 (소스 추가) (3분)
4. 직군별 첫 질문 던지기 → 결과 확인 (인용 번호로 원문 클릭) (8분)
5. 마인드맵 또는 오디오 개요 한 번 돌려보기 (3분)
```

> **인용 확인 강조**: "인용 번호를 클릭하면 원문으로 바로 점프 — 이게 Score 프롬프트보다 신뢰도 높은 이유"

## 4차시 실습 (E블록 · 13분)

### 기본 정보

| 항목 | 내용 |
|---|---|
| 도구 | Gemini (데이터 분석) + 리사(ReSa) |
| 프롬프트 | **리사(ReSa)** — 워크북 인쇄본 제공 |
| 방식 | 더미 CSV 업로드 → 인사이트 3줄 → 리사 보고 주제·대상·방식 입력 → 질문 답변 → 산출물 |
| 산출물 | 인사이트 3줄 + 보고 멘트 + 핵심 요약 3줄 + **PPT·구두·A4 중 1가지 목차** |
| 사전 준비 | 강사가 직군별 더미 CSV 1개씩 사전 배포 |

### 직군별 시나리오

| 직군 | 더미 데이터 | 리사 보고 유형 | 리사 방식 |
|---|---|---|---|
| 상품기획·오퍼레이터 | 분기 상품별 판매 데이터 | 현황 공유형 | PPT |
| 영업·판매 | 거래선별 매출 데이터 | 현황 공유형 | 1~2분 구두 |
| 예약·발권 | 노선별 예약률 데이터 | 판단 요청형 | 문서(A4) |
| 경영지원·인사총무 | 부서별 예산 집행 데이터 | 판단 요청형 | PPT |
| 고객서비스·VOC | 월간 VOC 카테고리 데이터 | 현황 공유형 | 1~2분 구두 |
| 카지노·VIP | VIP 등급별 이용 데이터 | 기획·전략형 | PPT |
| IT·시스템 | 시스템 장애·문의 데이터 | 판단 요청형 | 문서(A4) |

### 실습 흐름

```
1. 직군별 더미 CSV 다운로드 (1분)
2. Gemini에 CSV 업로드 → "임원 보고용 인사이트 3개 뽑아줘" → 인사이트 3줄 확인 (5분)
3. 리사 전문 붙여넣기 → 보고 주제·대상·방식 입력 (1분 30초)
   - 보고 주제: 직군별 데이터 주제
   - 보고 대상: 본부장 (또는 팀장)
   - 보고 방식: 직군별 지정 방식 (PPT / 구두 / A4)
4. 리사 질문 4~5개에 인사이트 3줄 활용해 답변 (4분)
5. 산출물 출력 확인 → 피드백 1회 (1분 30초)
```
