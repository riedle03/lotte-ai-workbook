> **총 슬라이드**: 18장 (표지 1 + 섹션 디바이더 1 + 콘텐츠 16)
> **표기법**: 헤드라인=H / 서브타이틀=S / 본문=B / 강사멘트=M / 레이아웃=L

- 카드 타이틀 (SemiBold 16pt): `1·2차시 — 만든 것`
- 내용 (Regular 13pt):
  - 1차시: ROADS + 프롬프트 마스터 → **뼈대**
  - 2차시: 기획방향성 마스터 + Tona + 카피 마스터 → **톤**
  - 공통: 내 머릿속 + 외부 지식으로 만든 결과물

**B 우열** (카드, #024ad8 테두리):

- 내용 (Regular 13pt):

  - NotebookLM → **내 자료 창고** (원문 안에서만 답)
  - 이 차시가 끝나면 → 1·2·4차시 결과물 품질이 한 단계 올라감

  무료 계정: 하루 약 3회 딥 리서치
  
  → 검색 결과마다 출처 번호 [1][2][3]
  → 각 번호를 클릭하면 원문 링크 이동
  → 복사해서 NotebookLM 소스로 바로 활용
  ```

**B 우열** (카드, #1a1a1a):

  Gemini   범용 작업·글쓰기 → 이번 강의 주력
  Perplexity  실시간 리서치 + 출처 → Deep Dive 전용
  
  → 이번 차시에서만 Perplexity 쓰는 이유:
    "어디서 왔는지 모르는 정보는
     보고서에 못 쓴다"
  ```

**하단 안내 박스** (#ff5050 테두리, #fff bg, radius 8px):
- (SemiBold 13pt, #ff5050): `⚠️ 지금 바로 확인 — Perplexity 접속 3단계`
- (Regular 12pt, #1a1a1a):
  ```
  1단계: 크롬에서 perplexity.ai 입력 → 엔터
  2단계: 오른쪽 상단 "Sign up" → 구글 계정으로 로그인 (30초)
  3단계: 검색창이 보이면 완료 ✓

  무료 한도: 일반 검색 무제한 / Pro 검색(딥 리서치) 하루 약 3회
  → 로그인 안 되셨으면 강사 화면 보면서 따라하세요
  ```

> "Perplexity는 AI 검색기예요.
> Gemini랑 다른 점은 — 출처가 붙는다는 거예요.
> 아무 정보나 AI가 만들어내는 게 아니라
> 실제 웹페이지를 검색해서 요약하고, 원문 링크를 붙여줍니다.
> 지금 perplexity.ai 들어가서 구글 계정으로 로그인해 두세요.
> 30초면 됩니다. 로그인 안 되셔도 제 화면 보시면서 따라오시면 돼요."

- 타이틀 (SemiBold 14pt): `강사 첫 질문`
- 내용 (Regular 12pt):
  ```
  "이 자료들을 바탕으로
   가을 동남아 가족 기획전의
   핵심 세일즈 포인트 5개를
   뽑아줘"
  ```
- 구분선 후 출력 예시 (Regular 11pt, #3d3d3d):
  ```
  ① 패밀리 전용 키즈 프로그램 [출처1]
  ② 선착순 28명 소규모 프리미엄 [출처2]
  ③ 부모 자유 시간 보장 구조 [출처1][출처3]
  ④ 3~9세 동반 최적화 일정 [출처3]
  ⑤ 전년比 패밀리 수요 +18% 대응 [출처2]
  ```

**B 우열 — 포인트 2가지** (카드, #1a1a1a):
- 포인트 1 (SemiBold 14pt, #fff): `[출처] 숫자 클릭 → 원문 바로 점프`
- (Regular 12pt, #fff 80%): 우측 패널에서 원문 해당 문장 하이라이트 — 이게 할루시네이션 방지의 실체

- 구분선 후 포인트 2 (SemiBold 14pt, #fff): `마인드맵 자동 생성`
- (Regular 12pt, #fff 80%):
  ```
  우측 상단 "노트북 가이드" → 마인드맵 탭
  3개 자료의 핵심 구조를 AI가 자동으로 시각화
  → 팀 보고 전 자료 구조 파악에 활용
  ```

**하단 So What** (#c9e0fc):
```
"이게 ② Check — 원문 안에서만 답하게 가둔 거다. 출처 없으면 답 못 한다."
```

**M**:
> "보이시죠? [출처1], [출처2] 숫자가 붙었어요.
> 이거 클릭하면 바로 원문으로 넘어갑니다.
> 내가 올린 PDF 몇 페이지에서 왔는지 — 바로 확인돼요.
> 그러면 이걸 보고서에 써도 됩니다. 틀린 말이 아닌 걸 확인했으니까요.

### S15 — 4차시 예고: 리사(ReSa)

**L**: Pattern F — Stacked (3개 차시 타일 → 예고 카드)

**B 상단 — 4개 차시 타일** (KPI 스타일, 가로 4등분):

타일 1 (#f7f7f7):
- 숫자: `1차시`
- 레이블: 뼈대 (ROADS + 프롬프트 마스터)

타일 2 (#f7f7f7):
- 숫자: `2차시`
- 레이블: 톤 (Tona + 카피 마스터)

타일 3 (#f7f7f7):
- 숫자: `3차시`
- 레이블: 창고 (NotebookLM 데이터 인프라)

타일 4 (#024ad8):
- 숫자 (#fff): `4차시`
- 레이블 (#fff 90%): **데이터 분석 + 보고 (리사)**

**B 중단 — 4차시 예고 카드** (#1a1a1a bg, radius 16px):
- 키워드 (Bold 20pt, #fff): `리사 (ReSa) — 보고 스토리라인 아키텍트`
- 내용 (Regular 13pt, #fff 80%):
  ```
  상사가 던질 질문을 AI가 먼저 설계한다
  → "보고자가 할 말"이 아니라 "상사가 궁금한 것"
  → 4가지 유형 진단 × 5가지 보고 방식
  → Gemini 데이터 분석 결과 → 리사 → 보고 멘트 + 핵심 3줄 + PPT 목차
  ```

**B 하단 — 준비물 박스** (#ff5050 테두리, #fff bg):
- (SemiBold 14pt, #ff5050): `⚠️ 4차시 시작 전 준비`
- (Regular 13pt):
  ```
  엑셀 또는 CSV 파일 1건 준비
  (본인 업무 관련 수치 데이터 — 더미 OK)
  지금 없으면 강사 제공 더미 파일 사용 가능
  ```

**M**:
> "1·2·3차시에서 뼈대·톤·창고를 만들었어요.
> 4차시는 그걸 하나로 합치는 시간입니다.
> 숫자 데이터를 Gemini가 분석하고 — 리사가 그 결과로 보고서까지 만들어줍니다.
> 잠깐 쉬고 시작할게요.
> 엑셀 파일 있으신 분 미리 열어두세요."
